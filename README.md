# 🗺️ Roadmap de Projetos

A estrutura segue uma progressão lógica baseada nas principais áreas da Segurança da Informação:

## 🔐 01 — Controle de Acesso (IAM)

Objetivo:
Implementar controle de acesso baseado em boas práticas de segurança.

Atividades:

- Criação de usuários e grupos no Linux
- Definição de permissões
- Princípio do menor privilégio
- Simulação de perfis (usuário comum e administrador)

Resultado:
Ambiente com controle de acesso estruturado e redução de riscos.

## 📊 02 — Monitoramento com Zabbix

Objetivo:
Implementar monitoramento contínuo de eventos.

Atividades:

- Configuração do agente Zabbix
- Monitoramento de CPU e memória
- Criação de alertas
- Análise de logs de login

Resultado:
Detecção de atividades suspeitas em tempo real.

## 🧠 03 — Análise de Logs (SIEM) (Em andamento)

Objetivo:
Centralizar e analisar eventos de segurança.

Atividades:

- Coleta de logs
- Correlação de eventos
- Identificação de padrões de ataque
- Geração de alertas

Resultado esperado:
Melhoria na detecção e resposta a incidentes.

## 🌐 04 — Segurança em Redes

Objetivo:
Proteger serviços e comunicação de rede.

Atividades:

- Análise de portas
- Controle via firewall
- Simulação de ataques (força bruta)
- Monitoramento de tráfego

Resultado:
Maior controle e visibilidade da rede.

## ☁️ 05 — Cloud Security (Básico)

Objetivo:
Aplicar conceitos de segurança em nuvem.

Atividades:

- Criação de máquina virtual (AWS/Azure)
- Configuração de acesso seguro
- Controle de portas

Resultado esperado:
Base sólida em ambientes cloud.

## 📜 06 — Governança, LGPD e Riscos

Objetivo:
Aplicar práticas de proteção de dados e conformidade.

Atividades:

- Classificação de dados
- Análise de riscos
- Definição de controles de acesso
- Simulação de cenário corporativo

Resultado:
Ambiente alinhado com boas práticas de governança.

## 🛡️ 07 — Hardening Linux

Objetivo:
Fortalecer a segurança do sistema operacional.

Atividades:

- Configuração de UFW
- Proteção SSH
- Implementação do Fail2Ban
- Restrição de acessos
- Monitoramento de logs

Resultado:
Sistema mais resistente a ataques.

## 🚨 08 — Resposta a Incidentes (Em desenvolvimento)

Objetivo:
Criar processos estruturados para resposta a incidentes de segurança.

Atividades planejadas:

- Criação de playbooks de resposta
- Identificação e classificação de incidentes
- Fluxo de resposta (detecção → contenção → erradicação → recuperação)
- Registro e documentação de incidentes

Resultado esperado:
Capacidade de resposta rápida e organizada a incidentes de segurança.

## 🛠️ Tecnologias e Ferramentas
